# Skills 設計

Claude Code のカスタムスキル・サブエージェントの設計を管理する。

## 方針

- CODING_GUIDELINES.md にサンプルコードがあるため、雛形生成系は通常の会話で対応可能（スキル化不要）
- スキルは **決まったコマンド列を毎回ブレなく実行する** 場合のみ作成
- コードレビューはサブエージェントとして実装

## スキル

| スキル名 | 概要 | ステータス |
|----------|------|-----------|
| post-impl-check | knip + similarity-ts による実装完了時チェック | 実装済み |

### post-impl-check

- **実行**: `/post-impl-check`
- **処理**:
  1. `bunx knip --no-progress` — 未使用エクスポート・ファイル・依存の検出
  2. `similarity-ts . --threshold 0.8` — コード重複の検出
- **対応方針**: 検出結果に応じて削除またはリファクタリングを提案。大きな変更はユーザー確認後に実施

## サブエージェント

| 名前 | 概要 | ステータス |
|------|------|-----------|
| review | CODING_GUIDELINES.md に基づくコードレビュー | 未実装 |

### review

- **目的**: CODING_GUIDELINES.md に基づくコードレビュー
- **入力**: 引数なし (ステージされた差分) または ファイル/ディレクトリ指定
- **チェック対象**:
  - クラス使用の有無 (ライブラリ境界以外)
  - 型アサーション (`as`) の使用 (`as const` は除外)
  - try-catch のルート以外での使用
  - neverthrow の Result 型パターンの遵守
  - 命名規則の遵守
  - コンパニオンパターンの適用
