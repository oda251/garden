# スキル・エージェント設計

Claude Code のスキルとサブエージェントの設計を管理する。

## 方針

- **スキル**: 固定コマンド列をブレなく実行する用途のみ作成
- **サブエージェント**: スキルの実行結果を判断し対応する
- CODING_GUIDELINES.md にサンプルコードがあるため、雛形生成系はスキル化不要

## スキル

| スキル名 | 概要 | ステータス |
|----------|------|-----------|
| post-impl-check | knip + similarity-ts による実装完了時チェック | 実装済み |

### post-impl-check

- **実行**: `/post-impl-check`
- **処理**:
  1. `bunx knip --no-progress` — 未使用エクスポート・ファイル・依存の検出
  2. `similarity-ts . --threshold 0.8` — コード重複の検出

## サブエージェント

| 名前 | 概要 | ステータス |
|------|------|-----------|
| coder | post-impl-check を呼び出し、結果に基づき修正 | 未実装 |
| reviewer | CODING_GUIDELINES.md に基づくコードレビュー | 未実装 |

### coder

- **目的**: 実装完了後の品質チェックと自動修正
- **トリガー**: 実装タスク完了時
- **処理**:
  1. `/post-impl-check` スキルを実行
  2. 検出結果に応じて自動修正。大きな変更はユーザー確認後に実施

### reviewer

- **目的**: CODING_GUIDELINES.md に基づくコードレビュー
- **入力**: 引数なし (ステージされた差分) または ファイル/ディレクトリ指定
- **チェック対象**:
  - クラス使用の有無 (ライブラリ境界以外)
  - 型アサーション (`as`) の使用 (`as const` は除外)
  - try-catch のルート以外での使用
  - neverthrow の Result 型パターンの遵守
  - 命名規則の遵守
  - コンパニオンパターンの適用
